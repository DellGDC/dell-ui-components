Product Stack Elements

Stack Label

The Stack Label is used to highlight a unique product attribute based on logic that incorporates legal requirements, lifecycle needs, timed events, and marketing differentiation.

Code/Logic

  • Optional. Allow ON only for items at SKU/OC level.
  • Automatically toggle OFF for items at Model and Franchise levels.
  • Text only. No special formatting.
  • Use an inheritance model to provide Bugs that can cascade across Franchise, Model, and OC levels.
  • In situations where there is no required legal need, display of the Bug is based on business logic and priority rankings. For example, business logic may specify that timed event Bugs can be overridden by ATS (available-to-sell) conditions, so that Cyber Monday Special text is replaced by Only [N] Left! when inventory levels hit a certain threshold.
  • Globalize as all other text per standard AppFramework globalization methods. Should read Left-to-Right or Right-to-Left based on language selection.

UX

  • Not clickable. No interaction.

Visual

  • Plain text only, formatted per visual comps. No links; no interaction.
  • Text is centered or left-aligned to match with other elements in the stack.
  • Text wraps as necessary.

Content Usage

  • Business-controlled copy assigned at the OC/SKU level. Content is ideally concise to keep the bug on one line, without wrapping.
  • The stack label can be displayed in any Product Stack context, including search results, Browse, product carousels, and Cart.
  • When the stack label is manually maintained for each Order Code, the Business must create a solid content strategy for assigning useful differentiators to each OC. When all OCs on a page show the same bug text (for example, “Dell Recommended”), the stack label no longer serves its purpose as a useful differentiator.
  • DCX recommends that the stack label be suppressed in Product Stacks for items whose Product Titlesare also using differentiator terms (such as Good, Better, or Best or Fast, Ultra Fast, or Turbo Charged). This is to allow the user to focus on one key differentiator per item and avoid any potential conflicts between differentiator content in the Bug and the Title.
