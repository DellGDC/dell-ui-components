Templates are in place for creating new components. All of the files needed to create a component are automatically placed in a directory that matches your component name.

How to create new component step by step

Step 1 Open a git bash.

Open a git bash in your work directory for dell-ui-components. To do this, you can click on the working folder and click Git Bash

If you don't see this you may need to install git so look in Getting Started section.

Step 2 Enter grunt command

First think of what the new component name is going to be. Make sure you dasherize the component name when entering command to create the new component. Example

$ grunt component:my-new-cool-component

You should see the following response:

$ grunt component:my-new-cool-component
Running "component:my-new-cool-component" (component) task
Copying files for my-new-cool-component component
Done copying files for my-new-cool-component component
Configuring my-new-cool-component component
Done configuring my-new-cool-component component

Done, without errors

Step 3 Check for new files

If you open your editor you should now see new files for the new component.

  1. HTML for demo of the new component. This HTML code will be copied into the Dell Design Library content management system (DDL).
  2. JavaScript in the demo controller which is strictly for demo purposes. This should be rarely used unless it is a complex component. If the demo is dependent on this JavaScript it will have to be put into the DDL for the demo to work.
    NOTE Only use this JavaScript file for demo purposes like providing the component sample data. If the component has functionality that requires new JavaScript then follow instructions for Adding Function to Components.
  3. CSS/LESS for demo purposes. Sometimes the CSS in the DDL conflicts with the package. A good example is components that are supposed supposed to go full width but inside the demo container they don't display properly. In the demo code you will want to use the same ID that is produced by DDL and adjust css with this ID so that changes are scoped properly.
  4. Currently not used. It is meant for building of component playgrounds in the future.
  5. Currently not used. It is meant as a placeholder for a component thumbnail.
  6. Component specific CSS/LESS. This is the CSS that gets included in the package
