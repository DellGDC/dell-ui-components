General

When you code HTML markup, think Twitter Bootstrap first. The following are helpful hints to help you code HTML in a clear way that leverages the most out of the Twitter Boostrap based dell-ui-components framework.

The container should NEVER be nested

There are no circumstances that ever warrant the use of a "container" inside another container. This can cause problems with grids inside the deeper container.

Good 
<div class="container">
	...
</div>
Bad 
<div class="container">
	...
	<div class="container">
		...
	</div>
</div>

Elements with grid classes col-**-**

  • Grid classes should only be applied to a div element except in the following:
    • buttons
    • table cells
    • form inut fields
  • Smaller breakpoint classes specify the colums for bigger breakoints. For example: col-sm-6 is the same as col-xs-12 col-sm-6 col-md-6 col-lg-6
  • Grid classes should never be accompanied by other classes except those cases above.
  • Grid classes should never be accompanied by custom classes
  • Grid elements should ALWAYS be inside a <div class="row"> element.
Good 
<div class="row">
	<div class="col-sm-6">
		...
	</div>
	<div class="col-sm-6">
		...
	</div>
</div>
Bad 
<div class="col-xs-12 col-sm-6 col-md-6 col-lg-6">
	...
</div>
<div class="col-xs-12 col-sm-6 col-md-6 col-lg-6">
	...
</div>

Text Blocks should always be inside paragraphs

  • <p></p> always provides the proper padding and margin according to style guide.
  • When paragraphs are not used you might find yourself creating a custom class to add margin or padding
Good 
<p>
	Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod
	tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam,

</p>

<div class="custom-class-that-provides-styling">
	Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod

</div>

Avoid the use of Custom Classes

  • Custom css classes easily introduce inconsistencies between different projects which.
  • Custom css classes make a project less usable by development teams relying on familiar standard Bootstrap code.
  • If you need a custom class that is altering margins or paddings you are probably doing something wrong such as:
    • Not using a paragrah tags around text block.
    • Not wrapping grid elements with a "row".
    • Not using dell-ui-components classes for top and bottom offsets
    • Not using well or panel to create visual containers
    • Not using dell-ui-components color branding classes.
  • If a custom class is a MUST, the less classes you use the better. Whenever possible, use ONE custom class to modify know Bootstrap elements. For example:
    Good 
    <div class="custom-class">
	<div class="alert alert-info">
	</div>
	<button class="btn btn-default">
	</button>
</div>	


CSS/LESS
.custom-class {
	...
	&.alert.alert-info {
		...
	}
	&.btn.btn-default {
		...
	}
}
    Bad 
    <div class="custom-class">
	<div class="alert alert-info custom-class-alert">
	</div>
	<button class="btn btn-default custom-class-button">
	</button>
</div>

CSS/LESS
.custom-class {
	...
	&.custom-class-alert {
		...
	}
	&.custom-class-button {
		...
	}
}
  • If a custom class is a MUST think Bootstrap when comming up with the name. For example:
    Good 
    <ul class="list-unstyled list-custom-name">
	...
</ul>
    Bad 
    <ul class="custom-name">
	...
</ul>
