Updating Sherpa Manager files with new package

By updating dell-ui-components in Sherpa Manager all projects can update their prototypes with the new package.

Step by step instructions

These instructions will show you what files you need to update but will not go into the git workflow of how to save these changes in the Sherpa Manager development environment.

Step 1 Get outside of the corporate firewall.

Bower updates use a particular transfer protocol and port to download files that is typically blocked by corporate firewall. The next step require this so you must have access outside of the corporate firewall.

NOTE If you are logged in through VPN you can skip this.

Step 2 Update bower package.

Open a git bash in your work directory for Sherpa Manager. To do this, you can click on the working folder and click Git Bash

$ bower update dell-ui-components

Step 3 Go back inside the corporate firewall

NOTE If you are logged in through VPN you can skip this.

Step 4 Update files for resources and heal function.

These files are used by Sherpa Manager to update existing projects with outdated themes. Make sure you replace these with the new package files.

NOTE You also need to update the files inside the zip folder which is used to create new prototypes. You will find the dell-ui-components folder inside projects/design-library.
