When creating new components all the files that are needed are automatically created for you except for any functional JavaScript files that support component function. The reason is that not all components require functional JavaScript.

Why controllers are not suitable for component function

  • Note that the standard set of files does include a JS file called demo-my-component.js. If you look in this file it already has a controller ready to go. You could actually write JavaScript inside this controller that would enable plenty of function. However, if you were to actually use this file, the resulting JavaScript would never get put into the dell-ui-components package during the build process.
  • In addition, the purpose of a controller in Angular should is to provide data to the scope. While it is fairly easy to write functions in controllers to to do things like manipulating the DOM in some way, controllers should only have data related business logic, not presentation logic. One of many reasons is because controller depends on an ng-controller reference in the HTML to work. That would require a lot of extra coding to get a component to There are other reasons why you don't want to use a controller for presentation functions and they are explained in Angular's documentation.

Functions with Angular dependency

It is preferable to build presentation functions in a non-Angular method. The reason for this is that not all applications supported by dell-ui-components are Angular applications. For this reason, it is preffered that component functionality is built as a jQuery plugin and then an Angular directive be built to be used in the demo and Angular applications.

Some component functions maybe put in place on an existing Bootstrap component that already has a jQuery plugin. A good example of this is Bootstrap's popover component. A dell-ui-components Angular directive has been built to autoimatically wire the popover function to popover elements in Angular applications using dell-ui-components.

In a non-Angular application, there would have to be a custom JavaScript function built to manually activate a popover element with Bootstrap's jQuery plugin.

Creating a plugin and directive to support an existing dell-ui-components component

Step 1 Open a git bash.

Open a git bash in your work directory for dell-ui-components. To do this, you can click on the working folder and click Git Bash

If you don't see this you may need to install git so look in Getting Started section.

Step 2 Enter grunt command

First, remember that you will be adding this plugin to an existing component. Now enter the following:

$ grunt plugin

NOTE You can also use an argument if you want the pluging name to be different that the component name.

$ grunt plugin:my-custom-plugin-name

You should see the following response:

Step 4 Understand the jQuery plugin template structure.

Take a look at your new dellUI.myCustomPluginName.js

Looking at the code above you should notice the usage of dellUImyCustomPluginName. If you are familiar with jQuery plugins you will understand that plugins are typically used on a jQuery selector.

Let's say you have a component which has a class name "foobar". Say that any element tagged with this class name should have a function alter it somehow. At the very minimum you would initiate the plugin as such:

$(".foobar").myCustomPluginName();

With options you would use something like:
$(".foobar").myCustomPluginName({option1:true,option2:false});

For more information about creating jQuery plugins visit the jQuery learn site.

Step 5 Wiring the jQuery plugin to the Angular demo via directive.

Take a look at your new my-custom-component.js (in the example above its called tabs.js)

The example above wires any element that has a class name of "my-custom-component" and executes the jQuery plugin on that element. You can build on this basic example to do more complicated things. Take a look at the tabs component.

For more information on directives, take a look at Angular directives documentation.

Creating an Angular Directive only.

In summary, the steps above will fit most of your needs. However, sometimes there might be a Bootstrap component that already has a jQuery plugin but youare seeking to somehow enhance it without altering the Bootstrap functionality. A good example of this is Bootstrap toggling functions that don't work properly in an Angular application because when the Bootstrap plugins fire the DOM may not be fully rendered.

If you look at the directives folder you will see such a directives file (toggle.js).

This directive helps an Angular application properly initiate Bootstrap elements that use the data-toggle attribute.

To create a directive, follow the steps above using a different grunt command

$ grunt directive:my-custom-plugin-name
