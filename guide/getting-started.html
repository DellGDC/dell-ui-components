Requirements

The development environment for dell-ui-components requires certain software to be installed in your computer. Some of these software packages require your computer to be outside of the corporate firewall to install.

Below are step by step instructions on how to set up the dell-ui-components development environment and all the software you require to get started.

Sublime Text is a free powerful text editor for the purpose of working with this development environment. Sublime Text also has the ability to be extended through a great number of plugins. If you already have Sublime Text you can skip to the next step.

Step 3: Install a Git GUI

While it is really beneficial to learn Git from the command line (git bash) some people just feel better with a GUI application that helps with git commands. There are two good options and you can install both if you want but at least one of these is a MUST:

Tortoise Git

Within Tortoise Git you can do pretty much anything that you do in the command line and more. Tortoise Git has a great diff (difference) editor that allows you to look and resolve merge conflicts.

Source Tree

Source Tree also allows you to do everything you can do in the command line and more but it has a better (more modern and visual) user interface than Tortoise Git.

Step 5: Check your environment path and access to node

At this point sometimes the path for node may not be properly configured. For this reason it is wise at this point to check from the command line first:

  1. Right click your desktop and you should see "Git Bash". If not, go back to step 2
  2. Once you see the command line, type npm and hit return.
  3. You should see a response from the Node Package Manager.
  4. If you see "command not found", your path variable may not be properly configured or you may not have NodeJS installed.
  5. If you know you have alrady installed NodeJS then check your path variables

Windows 7:

  1. Go to Start button and right click device.

  2. Click on advance settings

  3. Double click on PATH and confirm npm is in the path. If not, create the PATH variable as below.

  4. Repeat this step and if it fails, contact someone on the Sherpa team.
