Requirements

The development environment for dell-ui-components requires certain software to be installed in your computer. Some of these software packages require your computer to be outside of the corporate firewall to install.

Below are step by step instructions on how to set up the dell-ui-components development environment and all the software you require to get started.

Step 3 Install a Git GUI

While it is really beneficial to learn Git from the command line (git bash) some people just feel better with a GUI application that helps with git commands. There are two good options and you can install both if you want but at least one of these is a MUST. If you already have a Git gui installed you can skip to the next step.

Tortoise Git

Within Tortoise Git you can do pretty much anything that you do in the command line and more. Tortoise Git has a great diff (difference) editor that allows you to look and resolve merge conflicts.

Step 5 Check your environment path and access to node

At this point sometimes the path for node may not be properly configured. For this reason it is wise at this point to check from the command line first:

  1. Right click your desktop and you should see "Git Bash". If not, go back to step 2
  2. Once you see the command line, type npm and hit return.
  3. You should see a response from the Node Package Manager.
  4. If you see "command not found", your path variable may not be properly configured or you may not have NodeJS installed.
  5. If you know you have already installed NodeJS then check your path variables (need help?)

Windows 7:

  1. Go to Start button and right click device.

  2. Click on advance settings

  3. Double click on PATH and confirm npm is in the path. If not, create the PATH variable as below.

  4. Repeat this step and if it fails, contact someone on the Sherpa team.

Step 6 Get outside of the corporate firewall.

Node Package Manager uses a particular transfer protocol and port to download files that is typically blocked by corporate firewall. The next few steps require NPM so you must have access outside of the corporate firewall.

NOTE If you are logged in through VPN you can skip this.

Step 7 Install Grunt, Yeoman and Bower

Open a GitBash (right click on desktop and click Git Bash) and enter the following:

$ npm install -g grunt-cli yo bower

Step 8 Install CG-Angular Yeoman generator

Enter the following:

$ npm install -g generator-cg-angular

Step 9 Go back inside the corporate firewall

The next steps require access to internal servers.

NOTE If you are logged in through VPN you can skip this.

Step 10 Clone the Dell-UI-Components git repository

Open a GitBash (right click on desktop and click Git Bash) and enter the following:

$ cd c:

$ git clone http://dmvmsherpa.oldev.preol.dell.com:7990/scm/duc/dell-ui-components.git _duc

$ git checkout development

NOTE This will create a directory called _duc directly in your c drive. It is important to do this because windows has a limitation for the number of letters in a file's path so the shorter the directory path is the better.

Step 11 Get outside of the corporate firewall (again).

NOTE If you are logged in through VPN you can skip this.

Step 12 Install all NPM and Bower packages

Open a GitBash (right click on desktop and click Git Bash) and enter the following commands. You will see a lot happening after each install command. You may see some errors but for the most part these ca be ignored.

$ cd c:_duc

$ npm install

$ bower install

Step 13 Go back inside the corporate firewall

NOTE If you are logged in through VPN you can skip this.

$ grunt serve

Step 14 View development demo

Open your Chrome browser and go to http://localhost:9002/
